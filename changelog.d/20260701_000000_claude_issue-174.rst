Added
-----

- CSP violation reporting as a first-class Otto security primitive, pairing with
  the existing CSP emission (``enable_csp!`` / ``enable_csp_with_nonce!`` /
  ``Otto::Response#send_csp_headers``). ``Otto#enable_csp_reporting!(report_uri,
  &block)`` is the turnkey entry point: it appends a ``report-uri`` directive to
  every emitted policy (static and nonce), registers a violation callback, and
  injects ``Otto::Security::CSP::ReportMiddleware``. The middleware intercepts
  ``POST``\ s to the report path, parses BOTH the legacy
  ``application/csp-report`` and the Reporting API ``application/reports+json``
  wire formats into a normalized ``Otto::Security::CSP::Report``, enforces a
  64 KiB body cap (oversized bodies are dropped unparsed), tolerates
  malformed/empty input, and always answers ``204 No Content`` without touching
  application routes. It short-circuits ahead of the CSRF middleware, so
  browsers can post reports without a CSRF token, and a throwing application
  callback can never break the receiver. The lower-level primitives
  (``Otto::Security::Config#csp_report_uri=``, ``#on_csp_violation``,
  ``#dispatch_csp_violation``) are also available directly. When no report URI is
  configured, emitted CSP output is byte-identical to before. Otto normalizes
  reports but does not redact URL fields — applications decide logging/redaction
  policy in their callback. (delano/otto#174)
