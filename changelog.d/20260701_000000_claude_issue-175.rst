Added
-----

- ``Otto::Services``, a modular home for network-service integrations, with the
  Caddy on-demand TLS permission endpoint as the first one. Enable it with
  ``otto.enable_caddy_tls! { |domain| ... }`` — the block decides (truthy allows,
  falsy/nil/raise denies). One endpoint serves both Caddy's deprecated ``ask``
  directive and the ``permission http`` module. (delano/otto#175)

Security
--------

- The Caddy permission endpoint is loopback-only by default via
  ``Otto::Services::LocalhostGuard``: allowed only when the raw TCP peer is
  loopback and the request carries no forwarding header. Fail-closed on missing
  block, blank domain, and callback errors.
